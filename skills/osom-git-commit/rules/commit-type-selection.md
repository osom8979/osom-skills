# 커밋 Type 선택

| Type       | 용도                                          |
| ---------- | --------------------------------------------- |
| `feat`     | 새로운 기능                                    |
| `fix`      | 버그 수정                                      |
| `docs`     | 문서 변경                                      |
| `style`    | 포맷팅, 세미콜론 누락 등 (코드 동작 변경 없음) |
| `refactor` | 기능 변경 없이 코드 재구성                     |
| `test`     | 테스트 추가/수정                               |
| `chore`    | 유지보수 작업, 의존성 업데이트                 |
| `ci`       | CI 설정 변경                                   |
| `build`    | 빌드 시스템 또는 외부 의존성 변경              |
| `perf`     | 성능 개선                                      |

타입이 모호하면 변경의 **주된 의도**를 기준으로 선택합니다. 한 커밋에 여러 타입이 섞이면 커밋을 쪼개는 것을 우선 고려합니다.
