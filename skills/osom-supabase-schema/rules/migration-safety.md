# 마이그레이션 안전성

기존 데이터가 있을 가능성이 있는 테이블 변경 시 다음을 지킵니다.

- **컬럼 추가**는 `DEFAULT` 또는 `NULL 허용`으로 시작합니다.
- **제약 추가** 전에 기존 데이터의 정합성을 먼저 확인합니다.
- **파괴적 변경**(컬럼/테이블 삭제, 타입 변경)은 결과 보고에서 **"⚠ 마이그레이션 주의"** 로 경고합니다.

다단계 분리 패턴(권장):

1. 컬럼 추가 (`NULL` 허용 또는 `DEFAULT` 지정)
2. 백필 (기존 데이터 채우기)
3. 제약 추가 (`NOT NULL`, `CHECK`)
4. 구버전 제거 (다음 배포에서, 필요 시)

각 단계가 이전 배포와 호환되어야 무중단 배포가 가능합니다.
