# DB 함수 래핑 가능성

순수 쿼리로 해결 가능한 로직은 **DB 함수**로 구현해 Worker/Client에서 RPC 호출하는 편이 나은지 판단합니다.

DB 함수가 유리한 경우:

- 트랜잭션 경계가 단일 함수 호출 안에서 닫혀야 할 때
- 라운드트립을 줄여야 하는 다단계 쓰기
- RLS와 무관하게 권한이 격리되어야 하는 작업 (`SECURITY DEFINER`)
- 일관성 요구가 높아 클라이언트 락이 부적절할 때

반대로 단순 SELECT/INSERT는 클라이언트 SDK 호출이 더 명료합니다. 무조건 함수로 감싸지 않습니다.
